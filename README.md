# Fish Trajectory Forecast for Obstacle Avoidance System in AUV

This repository contains the complete implementation of a two-stage autonomous underwater navigation pipeline for Autonomous Underwater Vehicles (AUVs). The system utilizes a high-frame-rate **YOLOv11** object detector coupled with an encoder-decoder **Sequence-to-Sequence (Seq2Seq)** network to track and predict marine target trajectories for reactive obstacle avoidance.

---

## 📂 Project Structure

* **`Demo_Training_Data/`**: Contains baseline assets to verify data ingestion, coordinate smoothing, and target tracking.
* **`models/`**: Stores serialized model weights for both the object detection and sequential forecasting modules.
* **`Results/`**: Loss convergence graphs and multi-step inference trajectory visualizations.

---

## 🛠️ System Architecture

The pipeline processes raw optical frames and translates them into predictive coordinate vectors over two structural steps:

### 1. Object Detection (YOLOv11 Nano)
* **Model Weight:** `models/fish_yolo11n_best.pt`
* **Framework:** Ultralytics / PyTorch
* **Configuration:** Trained using the `yolo11m.pt` pretrained base model over 30 epochs with an image size of 416x416.
* **Optimization:** Unified with `single_cls=True` to collapse multiple marine classes into a single generic "object" bounding box, maximizing localization speed on resource-constrained embedded AUV hardware.

### 2. Trajectory Forecasting (Seq2Seq Network)
* **Model Weight:** `models/fish_seq2seq_triple_split.pth`
* **Framework:** Native PyTorch
* **Methodology:** Consumes spatial coordinate tracks passed from the detector and normalizes them into scale-invariant features. It uses an **Input Window (IW) of 20 time-steps** to output a **Forecast Window (OW) of 50 future steps** for real-time trajectory projection.

---

## 📈 Training Details & Loss Convergence

The vision module was optimized using the AdamW optimizer with an automatic learning rate scheduler on a Tesla T4 GPU footprint. 

### Performance Plots (`Results/Learning_Curve.png`)
* **Training MSE (Blue Line):** Shows a smooth, steep exponential decay, validating stable gradient descent and optimal parameter tuning.
* **Validation MSE (Red Line):** Demonstrates stable convergence with bounded, periodic oscillations, confirming that the model effectively navigates non-linear subsea movement dynamics without overfitting.

## 📂 Data Specification & Feature Mapping

### Ground Truth Logs (`Demo_Training_Data/B1_3_final_flip.csv`)
The sequential positional tracking profiles contain the following feature schema:
* `frame`: Incremental time-step sequence trackers.
* `x_raw` / `y_raw`: Raw pixel bounding box center-points generated by the YOLO pipeline.
* `x_smooth` / `y_smooth`: Filtered coordinates processed via spatial smoothing to mitigate subsea image jitter.
* `x_standardized` / `y_standardized`: Standardized spatial variables scaled to maintain uniform vector fields across variable object tracking distances.

## 🔮 Evaluation & Inference Result Verification

Multi-step forecasting evaluation outputs are saved as sequence validation matrices (`Results/Inference_IW_20_OW_50.png`). Individual simulation test-cases are mapped across three tracking profiles:

1. **Past (Dotted Grey Line):** The initial 20-step historical sequence fed into the encoder block.
2. **Actual (Solid Green Line):** The true ground-truth path traversed by the underwater target.
3. **Forecast (Dashed Red Line):** The 50-step look-ahead trajectory projected by the network to drive the AUV path-planning layer.
